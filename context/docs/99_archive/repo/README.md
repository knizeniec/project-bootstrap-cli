# Repository Archive

> **Purpose**: group archived repository-specific ADRs, specs, and plans under one navigable subtree.
> **Audience**: maintainers and auditors tracing historical repository decisions and working records.
> **When to update**: update when archived repository record locations or retrieval guidance changes.

This subtree keeps repository history that is no longer active guidance.

## Folders

- [adr/README.md](adr/README.md) for archived repository ADRs.
- [superpowers/README.md](superpowers/README.md) for archived repository specs and plans.

## Rule

Archived repository records stay here for traceability only. If one becomes active again, restore it to the active docs tree before treating it as current guidance.
